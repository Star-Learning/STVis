## 1. Episode Summary

Episode 001 is a cinematic 66-second animated visualization showing how the global economic center of gravity shifted from 1960 to 2024, told through country-level GDP share choropleth maps and a weighted-center marker.

### Key facts

- **Duration:** 66 seconds
- **Output resolution:** 2560Г—1440 (original), with 1920Г—1080 derivative
- **Data:** World Bank GDP (current US$) вҖ” indicator `NY.GDP.MKTP.CD`
- **Year range:** 1960вҖ“2024
- **Tech stack:** React + Vite + Framer Motion + Playwright + FFmpeg
- **Delivery:** Open-source data/scripts/viz + self-media-ready video

## 3. Data Processing Pipeline

### Step 1 вҖ” Download GDP data

```
scripts/01_download_world_bank_gdp.py
```
- Fetches `NY.GDP.MKTP.CD` from World Bank Indicators API
- Paginates through all countries and years
- Output: `data_raw/world_bank_gdp_raw.json`

### Step 2 вҖ” Clean and derive metrics

```
scripts/02_clean_gdp_data.py
```
- Keeps columns: `iso3`, `country`, `year`, `gdp_current_usd`
- Removes aggregate rows (world, income groups, regions)
- Derives per row:
  - `gdp_share` = country_gdp / global_gdp_that_year
  - `gdp_log` = log10(country_gdp)
  - `gdp_rank` = rank within year
  - `gdp_share_change` = year-over-year delta
- Output: `data_processed/gdp_country_year_clean.csv`

### Step 3 вҖ” Join with country boundaries

```
scripts/03_join_with_boundaries.py
```
- Downloads Natural Earth 110m country polygons via unpkg mirror
- Joins on ISO3 country code
- Output: `data_processed/world_gdp_joined.geojson`

### Step 4 вҖ” Generate render assets

```
scripts/04_generate_render_assets.py
```
- Builds compact `gdp_timeseries.json` with per-year country metrics and domains
- Generates `render_config.json` with animation timing and project metadata
- Computes weighted center and dominant continent per year

## 4. Visualization

### App

- React + Vite SPA, dark cinematic documentary style
- Config-driven: reads `render_config.json` and geometry + timeseries bundles
- State managed by Zustand
- Three visualization modes switchable in the UI:
  - `gdp_share` вҖ” share of global GDP (default)
  - `gdp_log` вҖ” log-scaled GDP
  - `gdp_share_change` вҖ” year-over-year change

### Visual layers

1. **Animated choropleth** вҖ” countries colored by selected metric
2. **Top-8 country halos** вҖ” highlight + rank labels for top economies
3. **Weighted center marker** вҖ” glowing dot moves as economic balance shifts
4. **Balance continent callout** вҖ” shows which continent dominates each year
5. **Side panel** вҖ” title, year, timeline scrubber, top-8 panel, mode toggle

### Export mode

- When `?export=1` is set in URL, the app exposes `window.__EPISODE_EXPORT__`
- `setElapsedMs(ms)` drives the animation deterministically
- `setMode(mode)` switches visualization mode
- Used by `export-frames.js` via Playwright for headless frame capture

## 5. Video Export Pipeline

```
scripts/05_export_video.py
```

1. Builds the viz app with `vite build`
2. Starts a preview server (`vite preview`)
3. Runs `export-frames.js` (Playwright headless) to capture 1989 PNG frames
4. FFmpeg assembles frames into MP4

### Key parameters

| Parameter | Value |
|---|---|
| FPS | 30 |
| Frame duration | 950ms per year step |
| Intro hold | 4000ms |
| Final frame hold | 1500ms |
| Total frames | 1989 |
| Total duration | ~66s |

## 9. Design Decisions

- **Why SVG instead of canvas/WebGL:** Better text rendering, simpler deployment, sufficient for country-level polygon data.
- **Why choropleth + weighted center:** A choropleth shows relative weight clearly; the weighted center adds a cinematic narrative anchor.
- **Why GDP share not raw GDP:** Raw GDP makes the map dominated by the US. GDP share shows relative importance and makes country trajectories comparable.
- **Why 1960вҖ“2024:** World Bank GDP data is complete enough from 1960 onward for a clean 65-year story.
- **Why 2560Г—1440:** Browser rendering requires a larger viewport to fit all UI elements; output is captured at native resolution then optionally downscaled.

## 10. Future Episode Expansion

The viz app is fully config-driven. Future episodes can reuse it by:

1. Replacing `world_gdp_joined.geojson` with a new country-level indicator geometry
2. Replacing `gdp_timeseries.json` with the new indicator's timeseries bundle
3. Updating `render_config.json` with new project metadata and animation timing
4. Swapping `default.yaml` config for new data paths

Possible Episode 002 indicators: population, COвӮӮ emissions, life expectancy, trade volume.
